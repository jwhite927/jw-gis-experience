(defn cycler []
  [:div
   {:class "flex items-stretch gap-3 mt-4"}
   [arrow-btn :prev]
   [project-card (current-location @selected-location)]
   [arrow-btn :next]])

(defn arrow-btn [dir]
  [:button
   {:class "w-11 grid place-items-center rounded-lg
            border border-stone-300 hover:border-accent"
    :on-click #(go! (if (= dir :prev) -1 1))}
   (if (= dir :prev) "‹" "›")])

;; cycling = pure index math on selected-location
(defn go! [dir]
  (swap! selected-location + dir)) ; current-location does the mod

⬢ map-view · mapbox interop

  • Your map-view form-3 already does this — keep create-map! in :component-did-mount, .remove in :component-will-unmount, render [:div#jw-map] (sized by the #jw-map rule in tailwind.css).
  • Swap .panTo → .flyTo for the eased camera: (.flyTo m #js {:center lnglat :zoom z :duration 1500 :curve 1.42 :essential true}).
  • Keep a marker vector so you can highlight: have create-marker! build a custom DOM el (click → (reset! selected-location i)) and collect the Marker objs in a defonce atom.

styling — tailwind v4 (css config)

  • You're on Tailwind v4 — config is CSS-based in resources/css/tailwind.css (@import "tailwindcss" + @source "../../src"). There is no tailwind.config.js. Utility classes stay in :class strings; lein dev rebuilds style.css.
  • If you want the hi-fi look, declare tokens in an @theme block (then bg-paper, text-accent, font-serif just work):
